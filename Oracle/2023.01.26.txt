DAO(Data Access Objects) 
  - 데이터베이스 데이터에 접근하기 위한 객체
  - 접근 로직 및 비지니스 로직을 분리하여 사용 

// 비즈니스 로직 : 입력, 수정, 삭제, 조회

DTO(Data Transfer Object) // 
  - 계층 간 데이터 교환을 위해 사용하는 객체
  - 로직을 갖지 않는 순수한 데이터 객체
  - getter / setter 메서드로 구성


메소드가 구성한 테이블과 똑같은 컬럼으로 구성되어 있음. -> update가 있다.
사용되어질 변수는 내가 DTO와 연결시킬 테이블의 컬럼명을  Privated변수로 사용
변수의 getter/ setter 로 ㅇㅇ
데이터베이스로 넘겨받은 자료, 사용자가 화면으로 받은 자료를 이동, 전송할 때 사용되어지는 객체



VO(Vlaue Object)
  - DTO와 유사한 객체이지만 read-only 특징
  - getter 메소드로만 구성
  - 값을 표현하기 위한 객체

오직 DB에 있는 자료를 꺼내서 읽을 때만 사용!
setter가 없고 getter만 존재


** 사용자가 데이터를 DB에 넣는 과정
  - 화면 또는 브라우져를 통한 자료 입력 -> 입력된 자료를 DTO 객체에 담아 DAO에 전달
    -> DAO에서 입력(또는 update) 로직을 사용하여 DB에 저장

// 직접 전달이 아니라 사용자가 입력받은 데이터를 담을 수 있는 객체
